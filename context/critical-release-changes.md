## High-Risk Rename Areas

### 1. npm workspace package names

These must be renamed together or imports/builds will break:

- `packages/schema/package.json`
  - `@launchkit/schema` -> `@baseforge/schema`
- `packages/generator/package.json`
  - `@launchkit/generator` -> `@baseforge/generator`
  - dependency `@launchkit/schema` -> `@baseforge/schema`
- `packages/templates/package.json`
  - `@launchkit/templates` -> `@baseforge/templates`
- `packages/shared/package.json`
  - `@launchkit/shared` -> `@baseforge/shared`
- `apps/web/package.json`
  - dependencies on `@launchkit/generator` and `@launchkit/schema`
- `packages/cli/package.json`
  - dev dependencies on `@launchkit/generator` and `@launchkit/schema`
  - smoke script that builds `@launchkit/schema` and `@launchkit/generator`
- root `package.json`
  - root package name `launchkit`
  - `test:smoke` script targeting `@launchkit/generator`
- `package-lock.json`
  - all workspace names and dependency links

Risk:

- Partial package rename will reproduce the same kind of module-resolution
  failure seen on Vercel.
- `package-lock.json` should be regenerated with npm after manifest changes.

### 2. TypeScript path aliases and framework build config

These are required for local TypeScript resolution and Next.js bundling:

- `tsconfig.base.json`
  - `@launchkit/schema`
  - `@launchkit/generator`
  - `@launchkit/shared`
  - `@launchkit/templates`
- `apps/web/next.config.ts`
  - `transpilePackages: ["@launchkit/generator", "@launchkit/schema"]`
- `packages/cli/scripts/build.mjs`
  - `launchKitWorkspacePlugin()`
  - plugin name `launchkit-workspace-source`
  - alias map for `@launchkit/generator` and `@launchkit/schema`
  - resolver regex `/^@launchkit\/(?:generator|schema)$/`

Risk:

- The CLI package bundles workspace source for publishing. If the package
  namespace changes but this esbuild plugin is missed, the published CLI build
  can fail or ship unresolved internal imports.
